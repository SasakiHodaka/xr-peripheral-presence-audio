# Research Story Brief

研究説明では、Scene Tokenから始めない。まず全体システムと利用者の困りごとを示し、その解決手段としてScene Tokenを位置付ける。

## 基本構成

```text
全体システム
  -> ユーザーストーリー
  -> 現状の課題
  -> 今回取り組む課題
  -> 提案手法
  -> 評価
```

## 1. 全体システム

対象は、工場で複数人がVR上で協調作業するシステムである。

```text
VR Factory Collaboration System
  -> 作業者
  -> 音声通信
  -> 空間音響
  -> 作業共有
  -> 状態共有
```

今回の研究は、この全体システムの中で「Scene Token生成」と「空間音響提示」を改善する部分に位置付ける。

## 2. ユーザーストーリー

```text
As a
VR工場で組立作業を行う作業員

I want
必要な情報だけを空間音声で受け取りたい。

So that
作業を止めずに、周囲の状況を把握しながら安全かつ効率的に協調作業できる。
```

例:

```text
作業者B「部品を持ってきて」
  -> Aは誰が話したか知りたい
  -> 右後ろから聞こえる
  -> すぐ反応できる

危険通知「フォークリフト接近」
  -> 重要な音として目立つ
  -> 事故を防げる
```

## 3. 現状の課題

現在の音声通信では、音声をそのまま送るだけになりやすい。

- 全音声を送信するため通信量が大きい
- 音の重要度を考慮できない
- 必要な情報と不要な情報を区別できない
- 重要な発話が埋もれる
- 状況把握や作業効率が低下する

## 4. 今回取り組む課題

本研究では、次の課題に着目する。

```text
VR工場での協調作業において、必要な情報だけを表現・伝送する情報表現が不足している。
```

つまり、音声波形をすべて送るのではなく、作業に必要な意味・空間情報だけを扱える形にする。

## 5. 提案手法

Scene Tokenを提案する。

```text
Scene Understanding
  -> Scene Token生成
  -> 通信
  -> 空間音響レンダリング
```

Scene Tokenは、次の情報を持つ。

- 話者
- 方向
- 距離
- 発話状態
- 重要度
- イベント

これにより、「誰が、どこから、どのように重要な情報を発しているか」を空間音声提示に利用できる。

## 6. 評価

評価では、Scene Tokenが通信量削減と状況把握支援に有効かを確認する。

- 通信量
- 話者定位精度
- 状況理解精度
- 重要発話への気づきやすさ
- 作業効率
- 主観評価

## 位置付け

Scene Tokenを主役にしすぎない。

```text
VR工場での協調作業支援システムがある。
その中に、通信量削減と状況把握の両立という課題がある。
その課題を解決するために、必要な情報だけを表現するScene Tokenを提案する。
```
